Code repository for PDP 2026 team 12 activity
## Proper repository maintenance 
* Commits should be well commented
* Commits need to pass unit tests before merging
* Commits can only be merged by the project manager
* New features will require new branches as well as unit tests
* Proper filesystem hierarchy must be used
* Proper licensing is to be used with the license as a single file in the repo
